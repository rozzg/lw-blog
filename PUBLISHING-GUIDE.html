Blog Setup &
Publishing Guide.

Your complete step-by-step for deploying the blog to Netlify, setting up the subdomain, and adding new posts every week.

Phase 1 · One time only
Deploy the Blog to Netlify
1
Create a GitHub repository
Go to github.com → New repository → name it lw-blog → set to Public → Create.
Why GitHub? Netlify deploys automatically every time you push new files to GitHub. No manual uploading needed.
2
Upload your blog files to the repo
In GitHub, click Add file → Upload files. Upload the entire folder structure:
lw-blog/
├── index.html
├── articles/
│ └── TEMPLATE.html
├── guides/
│ └── ai-tools-for-marketers.html
└── assets/
└── style.css
Commit message: Initial blog setup → Commit changes.
3
Connect the repo to Netlify
  • Go to netlify.com → Log in → Add new site → Import an existing project
  • Choose GitHub → Authorize → Select lw-blog
  • Build settings: leave blank (no build command, publish directory is / or leave empty)
  • Click Deploy site
Netlify will give you a random URL like magical-panda-123.netlify.app — your blog is live.
4
Set up the subdomain: blog.lifecycleworks.com
In Netlify (new blog site):
  • Site settings → Domain management → Add custom domain
  • Type blog.lifecycleworks.com → Verify → Add domain
  • Netlify will show you a CNAME record to add — copy it
In your DNS (where lifecycleworks.com is managed — likely Netlify DNS already):
  • Go to your main Netlify site → Domain settings → DNS settings
  • Add a new record: Type: CNAME · Name: blog · Value: paste the value Netlify gave you
  • Save. DNS propagation takes 5–60 minutes.
Already using Netlify DNS for lifecycleworks.com? Great — go to your Netlify team DNS panel, add the CNAME record for blog pointing to your new blog site's Netlify URL.
5
Enable HTTPS
In the new blog Netlify site → Domain management → HTTPS → Click Verify DNS configuration → then Provision certificate. This is free via Let's Encrypt and takes 1–2 minutes after DNS propagates.
6
Test everything
Open blog.lifecycleworks.com in your browser. Check:
  • Index page loads with your branding
  • Filter buttons (All / Articles / Guides) work
  • Post card links open the correct pages
  • Nav "Main Site →" links back to lifecycleworks.com
  • Looks correct on mobile
Phase 2 · Every week
Publishing a New Post
1
Duplicate the template file
Copy articles/TEMPLATE.html (or guides/TEMPLATE.html for guides). Rename it using a URL-friendly slug — no spaces, all lowercase, hyphens between words:
// Articles
articles/how-to-build-a-nurture-flow.html
articles/crm-audit-checklist.html

// Guides
guides/hubspot-workflow-guide.html
guides/email-segmentation-starter-kit.html
2
Fill in the 3 marked sections
The template has three clearly marked sections — look for the STEP 1 / 2 / 3 comments:
  • Step 1 — Update <title> and <meta name="description">
  • Step 2 — Update the cover: title, deck, date, read time, article vs guide tag
  • Step 3 — Write your post content between the <div class="post-body"> tags
Available content blocks inside post-body: <h2> section headings · <p> paragraphs · <blockquote> pull quotes · <ul>/<ol> lists · <div class="callout-box"> highlight boxes · <div class="post-divider"> visual breaks.
3
Add a card to index.html
Open index.html. Find the <!-- FEATURED --> comment and the existing cards. Copy one of the <a class="post-card"> blocks and paste it after the featured card. Update:
// 1. Update the href to your new file
href="articles/your-new-post.html"

// 2. Set the type for filtering
data-type="article" ← or "guide"

// 3. Update inside the card
tag: Article or Guide
date: Month Year
title: Your post title
excerpt: 2–3 sentence teaser
read-time: X min read
Make the newest post the featured card (full-width, top of grid) by moving it to the top and adding the class post-card--featured. Demote the previous featured card to a regular card by removing that class.
4
Upload to GitHub → Netlify auto-deploys
Go to your lw-blog GitHub repo:
  • Navigate to the correct folder (articles/ or guides/)
  • Click Add file → Upload files → Upload your new HTML file
  • Commit: Add: [post title]
  • Go back to root → click index.html → click pencil (edit) → paste updated index → Commit
Netlify detects the push and redeploys automatically. Your post is live in under 60 seconds.
Optional but recommended: Install the GitHub desktop app or use VS Code with Git for a smoother workflow — drag files in instead of uploading through the browser.
5
Update the filter count (optional)
The filter count updates automatically via JavaScript. No action needed — it counts visible cards on the fly.

Quick Reference
File Naming Conventions
→
Always use lowercase + hyphens for filenames
✓ Good
articles/will-ai-replace-marketers.html
guides/hubspot-onboarding-checklist.html

✗ Avoid
articles/Will AI Replace Marketers.html
guides/HubSpot_Onboarding.html
→
Path for assets is always relative
Files inside /articles/ or /guides/ link to the stylesheet with ../assets/style.css (two dots = go up one level). The index page uses assets/style.css (no dots, same level).
→
Article vs Guide — what's the difference
  • Article — opinion, perspective, story-driven. 400–900 words. Goes in /articles/. Use data-type="article" and tag--article / tag--dark-article.
  • Guide — how-to, framework, checklist, resource. 600–1500 words. Goes in /guides/. Use data-type="guide" and tag--guide / tag--dark-guide.
